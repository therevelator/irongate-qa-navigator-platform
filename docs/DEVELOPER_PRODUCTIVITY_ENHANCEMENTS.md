## DDPS Score Formula

### Overview

The **Daily Developer Productivity Score (DDPS)** is a 0-100 metric that measures developer efficiency using **absolute thresholds** rather than ratios. This ensures meaningful differentiation between productivity levels.

### Formula

```
DDPS = Focus Score + Meeting Penalty + Context Penalty + PR Bonus + Review Bonus + Balance Bonus
```

### Component Breakdown

| Component | Range | Calculation |
|-----------|-------|-------------|
| 🎯 **Focus Time** | 0 to +50 pts | 5h+ = 50, 4h = 40, 3h = 30, 2h = 20, 1h = 10 |
| 📅 **Meeting Penalty** | -10 to 0 pts | 0-1h = 0, 2h = -3, 3h = -6, 4h+ = -10 |
| ⚡ **Context Switch Penalty** | -15 to 0 pts | 0-2/day = 0, 3-4 = -5, 5-6 = -10, 7+ = -15 |
| 🔀 **PR Merge Speed** | -10 to +10 pts | <4h = +10, 4-8h = +5, 8-24h = 0, 24-48h = -5, 48h+ = -10 |
| 👀 **Review Speed** | -5 to +5 pts | <2h = +5, 2-4h = +2, 4-8h = 0, 8h+ = -5 |
| ⚖️ **Work-Life Balance** | 0 to +10 pts | ≤8h = +10, 9h = +7.5, 10h = +5, 11h = +2.5, 12h+ = 0 |

### Score Tiers

| Score Range | Status | Emoji |
|-------------|--------|-------|
| 80-100 | Exceptional | 🚀 |
| 60-79 | High | ⚡ |
| 40-59 | Healthy | ✅ |
| 20-39 | Low | ⚠️ |
| 0-19 | Needs Work | 🔧 |

### Key Design Decisions

1. **Focus Time is the Primary Driver** - Worth up to 50 points (50% of max score)
2. **Absolute Thresholds** - A developer with 1h focus time will always score low, regardless of other metrics
3. **Overtime-Based Work-Life Balance** - Penalizes working more than 8 hours, not focus/meeting ratios
4. **Meeting Penalty Capped at -10** - Reasonable penalty that doesn't overly punish developers in meeting-heavy roles

## New Derived Metrics

Four new derived metrics have been added to provide deeper insights:

### 1. Collaboration Score (0-100)
Based on code review responsiveness and PR turnaround time.
```
collaborationScore = 100 - (avgReviewTime * 2) - (avgPRTime * 0.5)
```
- ✅ ≥70: Excellent team collaboration
- ⚠️ ≥50: Good but could improve
- ❌ <50: Review and PR processes need attention

### 2. Burnout Risk Index (0-100%)
Calculates risk based on meetings, context switches, and low focus time.
```
burnoutRiskScore = (avgMeetingTime * 10) + (avgContextSwitches * 8) + ((8 - avgFocusTime) * 5)
```
- ✅ <30%: Low risk - healthy workload balance
- ⚠️ 30-60%: Moderate - monitor meeting load & context switches
- 🚨 >60%: High risk - reduce meetings & interruptions

### 3. Deep Work Ratio (0-100%)
Focus time as percentage of available work time (excluding meetings).
```
deepWorkRatio = (avgFocusTime / (8 - avgMeetingTime)) * 100
```
- ✅ ≥70%: Excellent focus time utilization
- ⚠️ 50-70%: Reasonable but can be optimized
- ❌ <50%: Too much fragmented time

### 4. Velocity Index (0-100)
Combined metric for PR and review speed.
```
velocityIndex = 100 - (avgPRTime + avgReviewTime) * 2
```
- ✅ ≥70: Fast code review & PR cycle
- ⚠️ 50-70: Moderate pace - optimize workflows
- ❌ <50: Slow cycle times affecting delivery

## AI-Powered Team Insights

### Overview

When a team is selected and AI is enabled for that team, the dashboard fetches AI-generated insights from the `/api/teams/:id/developer-ai-suggestions` endpoint.

### Features

- **"Powered by" Indicator**: Shows whether insights are from:
  - 🤖 **AI** - Generated by Groq API
  - 📊 **Rule-based** - Fallback algorithm when AI is unavailable
  - **Disabled** - AI not enabled for the team

### Developer Insights Card

For each developer, AI provides:
- **Status**: Healthy / At-Risk / Needs Attention
- **Summary**: Brief overview of developer's state
- **Strengths**: Key positive areas (max 2 shown)
- **Concerns**: Areas needing attention (max 2 shown)
- **Suggestion**: Actionable recommendation

### Enabling AI

AI is enabled/disabled per team in **Admin Panel → Teams → AI Toggle**.

## Work-Life Balance & Overtime Detection

### Overtime Calculation

Total daily work hours are estimated as:
```
totalWorkHours = focus_time + meeting_time + (code_review_time * 0.3) + (pr_merge_time * 0.2)
```

### Visual Indicators

Each developer card shows:
- **⚠️ Red overtime badge** when > 8h (e.g., "~9.2h/day (+1.2h overtime)")
- **✅ Green healthy badge** when 6-8h (e.g., "~7.5h/day")

### Work-Life Balance Scoring

The Work-Life Balance component of DDPS:
- ≤8 hours worked = **+10 pts** (100%)
- Each overtime hour subtracts **25%** (2.5 pts)
- 12+ hours = **0 pts** (0%)

## Bug Fixes

### 1. Team Balance Radar Chart - Efficiency Display

**Problem**: The "Efficiency" metric on the radar chart was showing 0.

**Cause**: The `avgEfficiency` value (DDPS score) was a 0-1 scale value, but the chart expected 0-100.

**Fix**: Multiplied `avgEfficiency * 100` when passing to `teamBalanceData`.

```typescript
// Before
{ metric: 'Efficiency', value: avgEfficiency, fullMark: 100 }

// After
{ metric: 'Efficiency', value: Math.round(avgEfficiency * 100), fullMark: 100 }
```

### 2. Team Selection Persistence on Reload

**Problem**: Selecting a team then reloading the page would reset to "all developers".

**Fix**: The `teamId` is now persisted in the URL as a query parameter (`?teamId=xxx`), and `App.tsx` reads it on load.
