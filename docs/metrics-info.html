Quality & Reliability Metrics

Test Coverage
test-coverage
Percentage of codebase covered by automated tests. Higher coverage reduces risk of undetected bugs.
Source
SonarQube → /api/measures/component?metricKeys=coverage
Formula
(Lines executed by tests ÷ Total LOC) × 100
Target
>80% critical, >70% overall
Test Flakiness Rate
flakiness-rate
Tests producing inconsistent results. Flaky tests erode CI/CD trust and waste developer time.
Source
Jenkins → track test reruns in testReport
Formula
(Flaky runs ÷ Total runs) × 100
Target
<2% (lower is better)
Defect Density
defect-density
Defects per thousand lines of code. Indicates code quality and predicts maintenance effort.
Source
Jira (bugs) + SonarQube (ncloc)
Formula
(Total bugs ÷ Lines of code) × 1000
Target
<0.5 per 1k LOC
Defect Escape Rate
defect-escape-rate
Bugs found in production vs. caught during testing. Measures testing effectiveness.
Source
Jira → filter environment=Production
Formula
(Production bugs ÷ Total bugs) × 100
Target
<5%
Code Quality Score
code-quality-score
Composite from static analysis. Identifies tech debt, security issues, maintainability.
Source
SonarQube → sqale_rating, reliability_rating, security_rating
Formula
Ratings: A=100, B=80, C=60, D=40, E=20
Weighted avg of 3 ratings
Target
>85/100 (A rating)

Speed & Efficiency Metrics

Average Build Time
build-time
Mean CI/CD pipeline duration. Faster builds enable quicker feedback loops.
Source
Jenkins → /job/{name}/lastBuild/api/json → duration
Formula
Sum(build durations) ÷ Build count (30 days)
Target
<10 minutes
Test Execution Time
test-execution-time
Total time to run automated test suite. Long times reduce deployment frequency.
Source
Jenkins → testReport/api/json → duration
Formula
Sum of test stage durations
Target
<30 min full, <5 min critical
Deployment Frequency
deployment-frequency
Production deploys per week. High frequency = mature DevOps. (DORA)
Source
Jenkins → deployment job builds[timestamp,result]
Formula
Count(successful prod deploys) ÷ Weeks
Target
>5/week (daily for elite)
Lead Time for Changes
lead-time
Time from commit to production. Shorter = faster delivery. (DORA)
Source
GitHub (commit time) + Jenkins (deploy time)
Formula
Median(deploy_timestamp - commit_timestamp)
Target
<1 day elite, <1 week high
Mean Time to Repair
mttr
Time to restore service after incident. Measures fix speed. (DORA)
Source
Jira → Bug tickets with resolved dates
Formula
Avg(resolved_date - detected_date)
Target
<1 hour elite, <1 day high
Parallel Test Efficiency
parallel-efficiency
How effectively tests parallelize. Better = faster total test time.
Source
Jenkins → wfapi/describe
Formula
(Sequential_time ÷ Parallel_time) ÷ Workers × 100
Target
>80% (near-linear)
Environment Startup
env-startup-time
Time to provision test environments. Faster = quicker test execution.
Source
Docker/K8s provision logs
Formula
Avg(ready_time - request_time)
Target
<5 min fast, <10 min ok

Agile & Process Metrics

Sprint Velocity
sprint-velocity
Story points completed per sprint. Predicts capacity for planning.
Source
Jira → /rest/agile/1.0/sprint/{id}/issue
Formula
Sum(story_points) WHERE status=Done
Target
Stable ±10% over 3-4 sprints
Sprint Commitment Rate
sprint-commitment
% of committed work completed. Indicates planning accuracy.
Source
Jira → sprint start vs end points
Formula
(Completed_points ÷ Committed_points) × 100
Target
>85%
Sprint Carryover
sprint-carryover
Work moved to next sprint. High = poor estimation or blockers.
Source
Jira → incomplete issues at sprint end
Formula
(Incomplete_points ÷ Committed_points) × 100
Target
<10%
First-Time Pass Rate
first-time-pass
Stories passing QA first attempt. Higher = better dev quality.
Source
Jira → workflow transitions
Formula
(Stories with 0 rejections ÷ Total) × 100
Target
>75%
Blocked Time
blocked-time
Hours tickets spend blocked per sprint. Identifies bottlenecks.
Source
Jira → /rest/api/3/issue/{key}/changelog
Formula
Sum(unblock_time - block_time)
Target
<15 hours/sprint
Automation Coverage
automation-coverage
% of test cases automated vs manual. Enables faster regression.
Source
TestRail → /api/v2/get_cases/{project}
Formula
(Automated_tests ÷ Total_tests) × 100
Target
>70% regression, >50% overall
Automation ROI
automation-roi
Return on automation investment. Justifies automation spending.
Source
Jenkins runs + time tracking
Formula
(Time_saved × Rate - Cost) ÷ Cost × 100
Target
>200%

Reliability & Stability Metrics

Change Failure Rate
change-failure-rate
Deploys causing failures/rollbacks. Measures deploy quality. (DORA)
Source
Jenkins + incident tracking
Formula
(Failed_deploys ÷ Total_deploys) × 100
Target
<5% elite, <15% high
Mean Time Between Failures
mtbf
Avg time system operates without failure. Higher = more stable.
Source
Datadog/New Relic monitors
Formula
Total_operational_hours ÷ Failures
Target
>100 hours
System Availability
availability
% time system is operational. Critical for SLA compliance.
Source
Monitoring uptime checks
Formula
(Total_time - Downtime) ÷ Total_time × 100
Target
>99.9% (three nines)
Infrastructure Failures
infra-failures
Test failures from infra issues, not code. Identifies env instability.
Source
Jenkins → failures tagged 'infrastructure'
Formula
Count of infra-caused test failures
Target
<5 per sprint

Composite QA Score

📊Overall QA Score Calculation

The QA Score is a weighted composite (0-100) representing overall quality health, combining four key indicators.

Component Weights

30%
25%
25%
20%
● Test Coverage (30%) ● Defect Escape Rate (25%) ● Build Success Rate (25%) ● Code Quality Score (20%)
Master Formula (from dataAggregator.ts)
QA_Score = Test_Coverage × 0.30
           + (100 - Defect_Escape_Rate) × 0.25
           + Build_Success_Rate × 0.25
           + Code_Quality_Score × 0.20
Example Calculation
Given: Coverage=85, Escape=4, Build=92, Quality=88

QA_Score = 85 × 0.30 = 25.5
           + (100 - 4) × 0.25 = 24.0
           + 92 × 0.25 = 23.0
           + 88 × 0.20 = 17.6
           = 90.1

Status Thresholds

≥85 → Good ✅ 70-84 → Warning ⚠️ <70 → Critical 🚨
